# Kylian Personal Site — Design Contract v4

## 这次纠偏解决什么

旧版把定位、方法和证据都做成了显性的编号、公式、卡片与指标条，阅读体验接近工作汇报或路演。新版不减少内容深度，但改变叙事方式：先让访客认识 Kylian，再自然地走进项目。

## 受众与任务

- 第一受众：招聘决策者、潜在合作者。
- 30 秒任务：看见一个具体的人，理解他在 AI 应用、Agent 评测与 GIS 交叉处工作，并愿意继续打开一个案例。
- 主要动作：阅读案例；次要动作：认识兴趣与联系 Kylian。

## 全站故事线

**观察一帧，拆开系统，亲手让它运转。**

这不是首页口号，而是组织视觉和内容的判断线：

- 摄影负责“观察”：人、空间、赛道与当下的真实光线。
- 项目负责“拆开”：把复杂问题变成能看见的流程与选择。
- 制作负责“运转”：原型、评测、AIGC 与桌宠都留下动手痕迹。

## 对标站吸收什么

- 单一而清楚的首屏意象，而不是多个卖点同时竞争。
- 白色纸面、深色主视觉、克制导航和内容优先的浏览节奏。
- 一个主题贯穿视觉、文字与动效。
- 大图、少量高质量栏目入口、自然的长页面节奏。

## 不复制什么

- 不复制星空、宇航员、IP 形象或商业社群结构。
- 不把首页做成知识库、电商落地页或内容转化漏斗。
- 不使用深色渐变卡片、粒子网络、玻璃拟态或 SaaS 仪表盘。
- 不再使用章节编号、价值公式、关键证据卡、三步方法条和四宫格能力卡。

## 视觉系统

- 62% 大理石白与暖白：页面、纸面和留白。
- 25% 午夜蓝与皇家蓝：主视觉阴影、标题和交互。
- 9% 铂金灰与淡蓝：分隔、图表背景和次级信息。
- 4% 旧香槟金：金属高光、细线和少量笔触。
- 中文全部使用舒适的无衬线字体；不使用宋体式中文标题。
- 页眉中的 Kylian 使用轻盈的手写签名字体，并以短金色收笔线结束；签名只承担个人识别，不扩散到正文。
- 容器使用有轻微不规则感的圆角矩形，不堆叠大面积空卡片。
- 赛璐璐感来自明确色块、轻微错版阴影、纸张颗粒与金色笔触，不在界面中直接画动漫人物。

## 页面节奏

### Home

1. 一张全宽个人照片 + 一句自然的人话 + 具体工作方向。
2. 一段“现在”：说明从 GIS 到 Agent 的内在线索。
3. 三个项目入口：像作品记录，不先展示指标。
4. 两个兴趣入口：AIGC 与桌宠。
5. 一段很短的 About 与联系。

### Work

- 先说明最近在处理什么问题。
- 每个项目由一张视觉封面、一个具体问题和一句价值解释进入。
- 实习经历作为一段真实经历，而不是 KPI 横幅。

### Project

- 为什么会做 → 真正的问题 → 关键选择 → 系统如何运转 → 我做了什么 → 证据与边界。
- 结构图和大图负责解释，卡片不承担全部叙事。

### Side Projects / About / F1

- Side Projects 是制作与好奇心。
- About 是人、教育和持续关注的事。
- F1 只作为 About 下的 Notes，不与 Work 同级。

## 动效尺度

- 首页照片有极轻的滚动位移，像镜头而不是产品发布会。
- 标题与段落按阅读节奏淡入，不使用整页蓝色转场。
- 项目封面 hover 只产生轻微抬升、错版阴影和图片位移。
- 鼠标反馈是一枚细金点与蓝色外圈；不遮挡内容。
- 所有动效遵守 `prefers-reduced-motion`。

## 验收问题

1. 首屏先看到 Kylian 这个人，还是先看到一套方法论？答案必须是前者。
2. 去掉姓名后，这个站是否仍能被任何产品经理套用？如果可以，个人线索还不够。
3. 项目入口是否让人想知道“他为什么做这个”，而不是先读 KPI？
4. F1 是否仍明显属于 About / Interest，而非职业能力证明？
5. 页面是否像个人编辑册，而不是公司季度汇报？
