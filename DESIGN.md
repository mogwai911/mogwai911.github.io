# Kylian 个人网站设计契约

## 目标

- 第一受众：招聘决策者与潜在合作方。
- 30 秒内让人看懂：Kylian 正在做 AI 应用落地、Agent 评测与工作流产品；优势是判断什么值得做，并把判断转成可验证的结果。
- 首页的首要动作：继续查看 Work，理解具体能做什么。

## 内容主线

一句话不是口号，而是降低理解成本：

> 判断什么值得做，然后把它做出来。

价值判断采用一个公开、易懂的表达：

> 是否值得做 = 预期影响 × 可干预性

Work 只收录具有共同主线的项目：Agent 失败诊断、信息分诊、GIS 智能工作流，以及真实组织中的高精地图实习。AIGC 漫剧与 Codex 桌宠放在兴趣作品；F1 Notes 作为 About 下的个人观察。

## 页面层级

```text
首页 Home
├─ 工作 Work
│  ├─ Agent 失败诊断
│  ├─ 信息分诊与学习助手
│  ├─ GIS 智能工作流
│  └─ 高精地图实习
├─ 兴趣作品 Side Projects
│  ├─ AIGC 多镜头创作
│  └─ Codex 桌宠
└─ 关于 About
   ├─ 个人介绍
   ├─ 教育
   ├─ F1 Notes
   └─ 联系方式
```

导航使用可折叠侧栏。首页不再保留大块栏目导航。

## 视觉

- 关键词：可信、好奇、复古外观、前沿内容、City Pop、泡沫时代杂志、赛璐璐硬边明暗。
- 不是动漫人物主题站，也不是企业科技官网。
- 白色：古希腊雕塑般的冷白石材，不是医院白。
- 蓝色：午夜蓝与皇家蓝承担结构和高对比内容面。
- 金色：旧金箔，只在章节编号、边缘和交互确认中出现。
- 构图：允许明显裁切、越界和杂志封面式大标题。
- 字体：现代无衬线；避免宋体、过细字重和大段粗黑体。
- 圆角：保留小到中等圆角，不能让所有内容都变成悬浮卡片。

建议权重：冷白 62%、蓝色 26%、淡蓝灰紫 8%、旧金箔 4%。

## 动效

- 鼠标移动应有明确反馈，但保留原生光标与可点击性。
- 页面反馈要流畅、有惯性；按钮反馈要干净、落点明确。
- Work 阅读区保持安静，动画只用于进入层级与交互确认。
- 页面切换使用蓝色遮罩与金色边缘；内容进入使用短距离位移和遮罩。
- 触摸设备和 `prefers-reduced-motion` 下自动关闭跟随光标、惯性和大幅转场。

## 明确不做

- 不造只有作者才能理解的新词或 slogan。
- 不做玻璃球、粒子星空、AI 网络、四宫格功能卡或产品发布会式首屏。
- 不让 F1 与 Work 同级，也不把兴趣包装成工作成果。
- 不使用难读的小字、过小界面截图、无意义大留白或轻飘悬浮反馈。
- 不公开真实姓名、电话、学生邮箱、本机路径、私有知识库或原始面试材料。

## 案例写作顺序

1. 谁遇到了什么问题。
2. 为什么值得解决。
3. 关键不确定性是什么。
4. Kylian 做了什么判断，为什么。
5. 哪些动作服务于判断。
6. 证据是否支持结果。
7. 方案边界和下一步。

案例首屏必须先呈现价值、角色和关键证据，细节随后展开。界面截图按原比例大图呈现，不并排缩小。
